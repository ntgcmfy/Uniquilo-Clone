==========================
MODULE 1: PRODUCT
==========================

1. Mục tiêu:
- Xây dựng API quản lý sản phẩm (CRUD) cho hệ thống bán hàng thời trang.
- Kết nối Supabase qua SDK, không truy vấn SQL trực tiếp.

2. Công nghệ sử dụng:
- NestJS (Node.js, TypeScript)
- Supabase SDK (@supabase/supabase-js)
- Biến môi trường: SUPABASE_URL, SUPABASE_ANON_KEY (lưu trong .env)
- Dotenv để load biến môi trường

3. Cấu trúc module:
- src/modules/product/product.module.ts
- src/modules/product/product.service.ts
- src/modules/product/product.controller.ts

4. Các API đã triển khai:
- GET /products           → Lấy danh sách sản phẩm
- GET /products/:id       → Lấy chi tiết sản phẩm
- POST /products          → Thêm sản phẩm mới
- PUT /products/:id       → Cập nhật sản phẩm
- DELETE /products/:id    → Xóa sản phẩm

5. Cách kết nối vào frontend:
- Frontend gọi các API trên qua fetch/axios (ví dụ: src/services/productService.ts)
- Đã thay thế hoàn toàn Supabase SDK ở frontend bằng API backend NestJS

6. Cách test backend:
- Dùng Postman/REST Client gửi request tới các endpoint trên
- Kiểm tra response trả về đúng dữ liệu từ Supabase
- Kiểm tra log backend nếu có lỗi

7. Những điểm reviewer có thể hỏi:
- Tại sao dùng Supabase SDK thay vì truy vấn SQL trực tiếp?
  → Để đảm bảo bảo mật, không cần quản lý kết nối DB, phù hợp với yêu cầu đề tài.
- Làm sao đảm bảo biến môi trường được load?
  → Sử dụng dotenv, kiểm tra file .env và import đúng ở supabaseClient.ts.
- Nếu Supabase bị lỗi thì API trả về gì?
  → Service sẽ throw error, controller trả về lỗi chuẩn NestJS.
- Làm sao frontend biết API backend đã hoạt động?
  → Test bằng Postman, kiểm tra response, kiểm tra hiển thị sản phẩm ở trang Home/ProductList.

8. Ghi chú thêm:
- Module này là nền tảng cho các module khác (cart, order, review, admin).
- Đã kiểm tra hoạt động với dữ liệu thật trên Supabase cloud.
